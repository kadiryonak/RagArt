RagArt — Kullanım Kılavuzu

Türkçe odaklı, plug-in mimarili bir RAG (Retrieval-Augmented Generation) sistemi. Aşağıdaki rehber her özelliğin ne zaman iyi olduğunu ve nasıl daha verimli kullanılacağını anlatır.

1. Workspace'ler nasıl çalışır?

Her workspace izole bir bilgi tabanıdır — kendi dosyaları, kendi vektör veritabanı. Farklı projeler için ayrı tutmak kalite ve gizlilik açısından kritik:

  • CV / özel evrak → kendi workspace'i (genel verilerle karışmaz)
  • Şirket dokümanları → ayrı workspace
  • Wikipedia örnekleri → Default workspace'te kalır
💡 İpucu: Dosya yükledikten sonra Bilgi Tabanını Yeniden İndeksle butonuna basmadan retrieval çalışmaz.

2. Retrieval stratejisi (Ayarlar)

StratejiNe zaman iyi?Ne zaman zayıf?
Dense (embedding) Anlamsal sorgular ("yapay zekanın faydaları"), paraphrase Özel isim/kod/kısaltma (BERT, FastAPI)
Sparse (BM25) Tam eşleşme, isim, kısaltma, nadir terimler Eşanlamlı / paraphrase
Hybrid (RRF) Her ikisinin avantajı — production default Çok küçük top-k'da gereksiz

Otomatik seçilirse hybrid kullanılır. Şüphedeyseniz auto bırakın.

3. Reranker (Cross-encoder)

İlk retrieval'dan gelen aday belgeleri ikinci bir model (query, doc) çifti üzerinden yeniden sıralar. Çok daha doğru ama yavaş (~6-8 saniye ekler).

  • Aç: Doğru cevabı bulamıyorsa, retrieval doğru kaynak getiriyor ama yanlış chunk seçiyorsa
  • Kapat: Hız öncelikli demo, yaygın faktöel sorular
İlk açtığında ~400MB bge-reranker-v2-m3 modeli indirilir (tek seferlik). Sonra cache'lenir.

4. Memory (Sohbet Hafızası)

Çoklu turn sohbet için. Tek soru-cevap senaryosunda none kalsın.

StratejiNe yapar?İdeal
Sliding window Son 5 turn'ü ham olarak prompt'a ekler Kısa sohbet, deterministik
Summary buffer Eskileri LLM ile özetler, son 4'ü ham tutar Uzun sohbet, token tasarrufu
Vector retrieval Geçmişten semantik olarak alakalı turn'leri çeker 50+ turn'lük uzun sohbetler

5. Context Engineering (Riskli — bilerek aç)

Eval baseline'larımızda top-k=5 + reranker konfigine bu işlemcilerin değer katmadığı görüldü (bkz. docs/eval-history.md v4). Default kapalı — ne zaman açın:

  • Dedup: Dense-only retrieval kullanıyorsanız (BM25 yok)
  • Reorder (lost-in-the-middle): 10+ chunk veya 100+ sayfa PDF
  • Token budget: API maliyeti kritikse

6. Provider & BYOK (Bring Your Own Key)

API anahtarınız sadece tarayıcınızda saklanır (localStorage), sunucuya gönderilmez. Her istekte X-API-Key header'ı ile geçer; sunucu hiç saklamaz.

Provider önerileri:

  • Groq — ücretsiz katman, llama-3.3-70b-versatile, hızlı. Test/showcase için ideal.
  • DeepSeek — ucuz, Türkçe iyi
  • OpenAI — gpt-4o-mini varsayılan, kalite üst düzey
  • Ollama — local, key gerekmez (yerel kurulum şart)
  • Local — fallback template, API key olmadan çalışır ama çok zayıf

7. Güvenlik — halüsinasyon koruması

Default'ta "Genel bilgi fallback'i" KAPALI. Retrieval boş döndüğünde LLM kendi bilgisinden cevap üretmez — bunun yerine "yeterli detay yok" mesajı döner.

⚠️ Açtığınızda LLM özel isim, kuruluş, proprietary içerik için UYDURMA CEVAP verebilir. Wikipedia tarzı kamu bilgisi soruyorsanız makul; özel/iç veride kapalı tutun.

8. Multi-format yükleme

Desteklenen: .json .pdf .docx .md .txt

  • PDF: Sayfa-bazlı chunk; metin katmanı yoksa (taranmış) çıkarım yapılamaz
  • DOCX: Paragraflar + tablolar (satır bazlı flatten)
  • Markdown: Code fence'ler ve frontmatter temizlenir
  • JSON: Liste / nested structure recursive extract

Yükleme sonrası Bilgi Tabanını Yeniden İndeksle butonuna basmadan dosya retrieval'a girmez.

9. Vektör Veritabanı seçimi

Workspace oluştururken seçilir, sonradan Ayarlar'dan değiştirilebilir (yeniden indeksleme gerektirir).

  • ChromaDB (default) — sıfır setup, embed-only mode, tek dosya, kolay backup
  • Qdrant — Rust-based, gelişmiş filter desteği. pip install qdrant-client sonra dropdown'da görünür
Daha fazla DB (Weaviate, Pinecone, Milvus, pgvector...) eklemek için BaseVectorStore'u implement edip VectorStoreFactory.register() çağırın — UI otomatik tanır.

10. En verimli kullanım — özet

  1. Workspace oluştur (proje bazlı izolasyon)
  2. API key gir (Groq önerilir, ücretsiz)
  3. Dosyaları yükle + Yeniden İndeksle
  4. Reranker'ı aç (yavaş ama doğruluk artar)
  5. Memory'i sohbet'e göre seç (tek soru → none)
  6. Halüsinasyon fallback'i kapalı tut (özel veri için)
  7. Cevap kaynaklarını tıkla — PDF'de doğru sayfa açılır

11. Sorun giderme

  • "Collection [UUID] does not exist": Yeniden İndeksle
  • "Kaynak bulunamadı" + cevap yok: Dosya yüklendi mi? Reindex'lendi mi? PDF taranmış mı?
  • Header fetch hatası (ISO-8859-1): Çözüldü (base64 encoding); sayfayı yenile
  • Reranker model indirme yavaş: İlk seferlik 400MB; sonra cache'li
